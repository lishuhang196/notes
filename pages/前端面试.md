- Web 会话追踪方法
	- Cookie，设置会话状态信息，每次用户请求，浏览器后自动将 Cookie 数据发送至服务器；
	- URL 重写，将会话标识追加到 URL 中作为查询参数的一部分，服务器通过解析 URL 中的会话标识追踪会话状态；
	- 隐藏表单字段，在表单中添加隐藏字段，用于存储会话标识；
	- Session，服务端设置 Session 存储当前会话状态
	- HTML5 Web 存储（storage api），客户端保存会话标识，每次用户请求都需要携带信息发送至客户端；

- 浏览器从输入 url 到页面展示中间发生了什么过程
  collapsed:: true
	- 解析 URL
	- 检查缓存
	  进入页面资源请求过程。浏览器进程会将 URL 请求发送至网络进程，通过网络进程发起真正的 URL 请求流程。
	  网络进程会先查找本地是否存在该资源缓存且缓存资源在有效期内，有这直接返回没有则进入网络请求流程
	  这一步中的缓存指的是强缓存
	- DNS 解析
	  获取域名对应的服务器 IP + Port
		- DNS 解析过程
		  本地 DNS 查询是一个递归的查询方式，分别自上而下查找
		  本地 DNS 服务器和根域名及其子域名是迭代查询方式
			- 浏览器缓存
			- 本地 hosts 文件
			- 本地 DNS 解析缓存
			- 本地 DNS 服务器
			  本地 DNS 服务没有查询到的话，则根据本地 DNS 服务器设置的转发器进行查询
				- 根域名服务器
				- 顶级域名服务器
				- 权威域名服务器
		- DNS 查询优化
			- DNS 解析也有缓存，一般会在本地记录解析记录。
			- 页面中尽可能减少使用过多域名。（HTTP 并发性受到源 限制，同一个源下一次最多并发 4-7 个）
			- DNS 预解析，<link rel="dns-prefetch" href="//[g.alicdn.com](http://g.alicdn.com/)"/>
			  当页面中引用了跨源资源（仅对跨域资源有效）时，可以使用dns-prefetch
	- 建立TCP 链接（ TCP/IP 协议），三次握手
	- 数据传输
	  TCP 连接建立完成后，浏览器端会构建请求行、请求头等信息，并将该域名下的 cookie 等数据附加到请求头中，向服务端放松请求信息。如果是 https，还需要进行 TSL 协商。
	  服务端接收到请求，解析请求头，如果存在 if-none-match和if-modifed-since，则校验缓存是否有效，有效则返回 304，无效则返回资源，状态码为 200；
	  这一步中涉及到 HTTP 中的协商缓存​
	- 关闭 TCP 连接（TCP/IP 协议），四次挥手
	- 浏览器
	  浏览器接受到资源后进行处理，如对 gzip 文件解压，解析 HTML 文档等

	- 预加载 & 懒加载
	  collapsed:: true
		- 预加载
		  preload 属性让浏览器提前加载指定资源（样式表、脚本等），并在需要执行的时候在执行，通常用于表示必定需要的资源（字体文件、首屏关键资源等）。
		  prefetch 属性（链接预取）是利用浏览器空闲时间加载页面将来可能用到的资源，通常用于加载其他页面（非首页）所需资源（异步模块）。
			- <link rel="preload" href="xxx.js" as="script">
			  collapsed:: true
			  link标签设置 rel 属性时必须设置 as 属性来声明资源类型
				- script
				- style
				- font
				- image
				- fetch，通过 fetch 或 XHR 获取的文件（ArrayBuffer、JSON 等文件）
			- <link ref="prefetch" href="">
		- 懒加载
		  延迟加载，在长网页中延迟加载图像，等到元素滚动到可视范围内时在加载。
		  根源便是减少网络请求
			- 图片懒加载
				- 实现原理
					- IntersectionObserver 重叠观察者，用于判断两个元素是否重叠
					- 滚动监听
			- 路由懒加载
			  动态加载模块方法import()
			- 组件懒加载
			  动态加载模块方法 import()
			- 优点
				- 减少无效资源请求，减少服务器负载
				- 使网页加载速度更快，提升用户体验
				- 防止并发加载资源过多阻塞js 加载（同一源下http 最大并发数 4-9）
- 如何实现夜间模式

	- 如何使用 CSS实现夜间模式
	  ```css
	  /* 媒体查询 */
	  @media (prefers-color-scheme: dark) {
	    /* css 规则 */
	  }
	  ```
	- `window.matchMedia("(prefers-color-scheme: dark)")`方法监听系统主题变化
	  ```javascript
	  const initTheme = (arg) => {
	      let scheme;
	      if (!arg) {
	          scheme = window.matchMedia('(prefers-color-scheme: dark)');
	          // 监听 mode 变化
	          scheme.addEventListener("change", initTheme);
	      } else {
	          scheme = arg;
	      }
	      console.log(scheme);
	  };
	  
	  initTheme();
	  ```
