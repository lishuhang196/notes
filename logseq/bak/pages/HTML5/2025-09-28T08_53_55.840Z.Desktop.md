		  meta 标签由name和content属性定义，用于描述文档属性，如网页作者、描述、关键字等。
			- 关键字描述<meta name="keywords" content="arg1,arg2">
			- 页面描述<meta name="description" content="arg1,arg2">

		- script 标签
		  普通script标签会阻塞浏览器对 HTML 的解析
			- async 和 defer 属性区别
			  async 异步下载脚本，不会阻塞浏览器解析 HTML，JS 脚本下载完成后，如果 HTML 还没有解析完毕，则先执行 JS 代码，执行完成后在继续解析 HTML。async 执行时机不可控，多个 async script 执行属性也不确定，async 脚本在页面 load事件之前发生，但可能在DOMContentLoaded之前或之后，因此不应该在 async 脚本内同步执行 DOM 操作。
			  defer 异步下载脚本，不会阻塞浏览器解析，JS 脚本下载完成后，如果 HTML 还没解析完成，等待 HTML 解析完成在执行 JS 代码。在 HTML5 规范中，多个 defer script 按照声明的执行顺序执行，且都会在DOMContentLoaded事件之前执行。
			  DOMContentLoaded&load事件
			- type="module"用于加载 ES6 模块
			  对于type="module"加载的 JS 脚本，都是异步加载的，不会造成浏览器堵塞，等到页面渲染完毕后在执行。等同于打开了 script 标签的 defer 属性
			  多个type="module"script 标签，会按照声明顺序执行。
			  且都会在DOMContentLoaded事件之前执行。

		- a 标签

		  发短信<a href="sms:110">110</a>"​
			- href="javascript:void(0)"和href="#"区别
			  href="#"表示锚点，页面会自动滚动到对应锚点位置（与元素id属性对应，默认是网页顶部）。
			  href="javascript:void(0)"死链接，用于阻止默认事件（阻止页面跳转）。
			- target="_blank"属性
			  通常是打开一个新的标签页（类似于window.open）
			  通过此方式打开的子页面可以通过window.opener获取到父页面的句柄，控制父页面
			  可用通过
			  <a href="xx.html" target="_blank" ref="noopener noreferer" >跳转</a>
			  var son = window.open("xx.html"); son.opener = null;

			  用于嵌入一个网页
			- 优点 & 缺点

			  可以将原有页面嵌入；

			  iframe 会共享连接池，影响页面并行加载。

				  DENY 禁止所有页面在 iframe 中嵌入，无论来源域名是什么。
				  SAMEORIGIN 允许同源页面在 iframe 中嵌入。
				  allow-from [XXXX.com](http://xxxx.com/) 允许指定域名网站嵌入

- 浏览器渲染

	- 渲染原理
	  collapsed:: true
	  ![](https://document-image.mubu.com/document_image/2087032_cd45c50d-994b-4d28-ee7a-fb8b3d7c9128.png)
		- 解析阶段
		  collapsed:: true
			- HTML 解析并构建 DOM 树
			  HTML 解析遵循自上而下的顺序解析，解析过程中遇到外联资源（link、script、iframe、image、video）会立即请求对应的资源
				- 阻塞性请求资源
				  立即停止解析当前 HTML 文档，请求并解析对应资源，完成后继续解析 HTML
					- 内联 JS
					- 外联 JS
					- 外联 async JS 资源
					  async script 网络请求是异步的，不影响 HTML 解析，资源加载完成后，如 HTML 还没有解析完毕，则停止解析 HTML，开始解析 JS 脚本。
					  ​多个async声明的外部文件，不保证执行顺序
					  ​window.load事件之前执行
					- 内联 CSS 资源
					- iframe
				- 非阻塞性请求资源
					- 外部 CSS 资源
					  CSS 资源加载不会阻塞 HTML 的解析，但会阻止渲染生成
					- image 资源
					- 外联 defer JS 资源
				- 关联知识点
					- async 和 defer 属性区别
			- 构建 CSSOM 树
			  解析 CSS 文件，构建 CSSOM 树
		- 构建渲染树
		  此阶段将上一阶段的 DOM 树和 CSSOM 树构建为 Render 树。Render 树以 DOM 树为基础，遍历 CSSOM 树（根据）生成，最终获得每个 DOM 节点的层叠样式。
		- 布局（Layout）阶段
		  Layout 阶段会生成第四颗树（呈现树），此阶段会获得每个元素所在浏览器视口的几何数据（位置、大小等）。
		  浏览器默认为流式布局，布局从上至下，从左至右依次解析。
		  布局过程会排除非视觉、display:none等节点，计算元素位置信息，构建一颗只包含可见元素的布局树
		- 绘制（Paint）阶段
		  此阶段渲染引擎会根据呈现树生成一颗对应的图层树（Layer Tree，根据3D 转换，页面滚动、z-index 等），分别为不同图层绘制。
		  生成一系列绘制指令
		- 渲染（Display）阶段
		  将上一阶段的绘制指令进程光栅化处理（将图块转为位图）。最后由合成线程将各个图层生成的图片合成，最终发送到后缓冲区。
	- 页面重排（回流）和重绘
	  collapsed:: true
	  现代浏览器会维护一个队列，将所有引起回流和重绘的操作放入队列中，如果队列中的任务数量或时间间隔达到阈值就会触发清空队列的操作。在访问元素部分属性或方法时（client-*、offset-*、scroll-*、width、height、getComputedStyle、getBoundingClientRect），就会强制触发清空队列
		- 重排（回流）
		  发生在布局阶段，元素布局位置信息发生改变时，浏览器需要重新计算元素的位置和大小
		  触发重排，一定会触发重绘
			- 元素增加或删除
			- 修改元素盒模型
			- 修改元素尺寸
			- 修改布局属性（margin、padding、border、position）
			- 浏览器窗口大小发生改变
			- 元素字体发生改变
			- 查询元素某些属性或方法（clientWidth、clientHeight、clientTop 、getComputedStyle、getBoundingClientRect等）
		- 重绘
		  重绘阶段，元素外观发生变化时，浏览器会重新绘制元素的过程
			- 元素背景、颜色、visibility等发生改变
		- 优化
			- 避免使用 CSS 表达式（calc）
			- 避免 DOM 结构层级过深
			- 避免使用 table 布局
			- 避免频繁操作大量 DOM
				- 通过 DocumentFragment 操作 DOM
				- 通过display:none
			- 避免频繁操作样式，通过修改 class 或者一次性写入 style 样式
			- 动画：
				- requestIdleCallbakc 代替 setTimeout
				- 动画尽量使用在absolute或fixed的元素上；
